Teaching

Deep Learning for Software Engineering: Taught both in seminar style for advanced graduate students, and at the Master's level. The goal is to rapidly acquaint students with general deep learning methods and specifically their application to software engineering. Teaching is greatly helped by hands-on experience, in which students are provided with models and data of recent published work. Assignments and/or course-long project then involve implementing and extending existing state-of-the-art published tools, typically in small teams.

Software Engineering Toolset: My former advisor and I constructed this course at UC Davis, which rapidly introduces a wide span of software engineering tools -- think everything from unit testing, version control, configuration management, static analysis, continuous integration, all the way to Docker + cloud deployment. Each is taught through both through their theoretical foundations and background, and in weekly hands-on discussion sections. This course was quite popular among CS students, for one because it provides a window into the many tools and aspects of real-world software engineering, but even more so (I hope) because it teaches the ability to quickly adopt, contextualize and understand new tools. I think that ability is pivotal to good software engineering, much more so than knowing any one tool by heart.
